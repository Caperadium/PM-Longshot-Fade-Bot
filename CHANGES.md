# Changes

- Port the VPS-local IPv4-egress fix (commit df5bd58, previously only on the VPS) onto the refactored engine so the VPS can fast-forward to origin/master: new `fader/infra/ipv4.py` (`force_ipv4()` patches urllib3's `allowed_gai_family` so all requests/urllib3 traffic including py-clob-client and Telegram resolves IPv4-only), wired into engine startup (`engine/main.py`, after `setup_logging`) and dashboard load (`dashboard/app.py`); `marketdata/ws_client.py` passes `family=socket.AF_INET` to `websockets.connect()`. Rationale unchanged from the original commit: Polymarket's Cloudflare-fronted hosts serve AAAA records but IPv6 egress to Cloudflare is unroutable on the OVH VPS, so getaddrinfo's IPv6-first ordering stalled the WS handshake and slowed every REST call. Process-scoped; no system config touched.
