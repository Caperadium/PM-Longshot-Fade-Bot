# Changes

- Fix WS watchdog reconnect churn on quiet feeds ("WS reconnected" telegram spam, observed as a 110s force-reconnect loop on the VPS overnight). Three parts: (1) the feed-silence watchdog no longer force-closes a socket whose PONGs are fresh (a quiet market is not a half-open socket; a server that never PONGs keeps the old behavior); (2) the "WS reconnected" telegram alert is throttled to one per 10 minutes, with suppressed reconnects counted and reported in the next alert (`alert_ws_reconnect` gains an optional `suppressed` arg); (3) tokens whose REST /book resync 404s three consecutive times are unsubscribed as gone (resolved/expired markets were 404-ing on every resync — 604 requests/30min observed), via a new `BookNotFound` exception from `fetch_order_book_snapshot`, and `_get` no longer retries deterministic 4xx responses. Files: `fader/marketdata/ws_client.py`, `fader/marketdata/rest_market.py`, `fader/infra/telegram.py`, `fader/tests/test_stability_fixes.py`.

- Make model-pricer decisions distinguishable from naive on the dashboard (`fader/dashboard/app.py`). Decisions tab: parse `model_p_yes`/`model_edge_no` out of `filters_json` into real columns, derive a per-row `model` verdict (`VETOED` = rejected with reason `model_edge_low`; `agree`/`would-veto` = entered with model edge at/below the CURRENT `pricer.min_edge`; `naive` = no model opinion recorded), add summary metrics (agree / would-veto / naive / vetoed counts) plus `MODEL VETOED` and `MODEL WOULD-VETO` filter options, and a caption showing live veto state + min_edge. PnL tab: new "Model Pricer Split" table grouping closed positions' realized PnL (trades, total, avg, hit rate, avg model edge) by the verdict recorded on each slug's entry decision -- the earns-its-keep readout for deciding when to flip `pricer.veto` on. Verdict helper is NaN-aware (pandas coerces None-bearing float columns to NaN; a NaN model edge must read as `naive`, not `would-veto`). Verified via extracted-helper unit assertions and a headless `streamlit.testing.v1.AppTest` full-render pass (9 tabs, no exceptions).

- Wire the vendored FIGARCH pricer (repo-root `core/pricing/`, see PRICER_README.md) into the live engine as an optional model-edge signal for BTC daily binaries. New `fader/strategy/model_pricer.py` (`ModelPricer` + `parse_btc_market`): per-expiry ladder cache (one `calculate_probabilities` call per expiry per `reprice_s`), background compute on the shared executor with single-flight + retry backoff, BTC-data staleness gate, optional self-refresh via the vendored stdlib `core/data/data_fetcher.py`, calibrated jump-param loading with the lam->lambda key mapping, and GARCH-cache refit cadence. Strategy loop consults it after filters 1-8: log-only mode (default, `pricer.veto: false`) attaches `model_p_yes`/`model_edge_no`/`model_age_s` to entered decisions' filters JSON; veto mode rejects with reason `model_edge_low` when the model NO edge is below `pricer.min_edge`. Fail-open everywhere: any missing model opinion (disabled, non-BTC slug, no cache yet, stale data, engine failure) leaves naive behavior untouched. New hot-reloadable `pricer` config section (config.yaml + `PricerConfig` in config_loader, all fields hot); `build.py` constructs and injects the pricer. 19 new tests (`fader/tests/test_model_pricer.py`), full suite 454 green; end-to-end smoke against the real engine verified from the repo root. Rationale (v3 backtest, NO band 0.55-0.95): filter is modest on this band (edge>=0 keeps 66% of trades at +7.3c vs naive +6.5c) and post-feb-2026 evidence is mixed, hence log-only default to accumulate live evidence before enabling veto. NOTE: the vendored engine itself (`core/`, `PRICER_README.md`) is gitignored -- this repo is public -- and deploys to the VPS out-of-band via scp.
