# Changes

- Fix WS watchdog reconnect churn on quiet feeds ("WS reconnected" telegram spam, observed as a 110s force-reconnect loop on the VPS overnight). Three parts: (1) the feed-silence watchdog no longer force-closes a socket whose PONGs are fresh (a quiet market is not a half-open socket; a server that never PONGs keeps the old behavior); (2) the "WS reconnected" telegram alert is throttled to one per 10 minutes, with suppressed reconnects counted and reported in the next alert (`alert_ws_reconnect` gains an optional `suppressed` arg); (3) tokens whose REST /book resync 404s three consecutive times are unsubscribed as gone (resolved/expired markets were 404-ing on every resync — 604 requests/30min observed), via a new `BookNotFound` exception from `fetch_order_book_snapshot`, and `_get` no longer retries deterministic 4xx responses. Files: `fader/marketdata/ws_client.py`, `fader/marketdata/rest_market.py`, `fader/infra/telegram.py`, `fader/tests/test_stability_fixes.py`.
