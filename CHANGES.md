# Changes

- Add Calibration dashboard tab + engine auto-fetch poller to verify the longshot bias still exists in current market data. New `fader/backtest/calibration.py` (pure core: `build_observations` one-obs-per-market at fixed DTE, `bucket_calibration`, `band_summary`, `monthly_edge`, `bot_trade_calibration`, hand-rolled Wilson CIs) and `fader/dashboard/calibration_page.py` (9th tab, embedded + standalone; implied YES prob vs actual YES resolution rate, per-bucket table, monthly edge trend, bot-realized-calibration section; introduces `st.cache_data` mtime-keyed snapshot load, a net-new pattern in the dashboard). Engine side: `fader/backtest/historical.py` gains atomic `ContractPriceStore.save()` (tmp + `os.replace`, retry on Windows lock), module-level `_apply_fetch_result`, pure helpers `select_stale_unresolved`/`_series_scan_start`, and incremental `update_calibration_data` (recent-window series discovery + re-fetch of stored-but-unresolved markets past end_date, closing the skip-existing resolution gap); new `Pollers._calibration_loop` (runs at startup then every `polling.calibration_fetch_s`, default 21600s, 0 disables, telegram escalation after 5 consecutive failures); `polling.calibration_fetch_s` added to `PollingConfig`, hot-reload, and config.yaml. Also corrects the `save_resolution` docstring (falsely claimed the backtest engine joins resolutions.csv; mechanism is unused). The tab uses the live engine's EFFECTIVE band and DTE window: `load_config()` + `apply_config_kv_overrides(cfg)`, so dashboard-written `config_kv` overrides (which shadow `strategy.band_low`/`band_high` etc.) are reflected, with a caption naming the band source (config.yaml vs live override). Tests: `fader/tests/test_calibration.py` (30) + `fader/tests/test_calibration_update.py` (17); full suite 427 passing.

- Port the VPS-local IPv4-egress fix (commit df5bd58, previously only on the VPS) onto the refactored engine so the VPS can fast-forward to origin/master: new `fader/infra/ipv4.py` (`force_ipv4()` patches urllib3's `allowed_gai_family` so all requests/urllib3 traffic including py-clob-client and Telegram resolves IPv4-only), wired into engine startup (`engine/main.py`, after `setup_logging`) and dashboard load (`dashboard/app.py`); `marketdata/ws_client.py` passes `family=socket.AF_INET` to `websockets.connect()`. Rationale unchanged from the original commit: Polymarket's Cloudflare-fronted hosts serve AAAA records but IPv6 egress to Cloudflare is unroutable on the OVH VPS, so getaddrinfo's IPv6-first ordering stalled the WS handshake and slowed every REST call. Process-scoped; no system config touched.
